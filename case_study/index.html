Microinteractions Case Study

Overview:

For this project i was tasked with developing three microinteractions varying in difficulty (simple, medium, complex) for the IDM 241 Microinteractions course at Drexel University. The goal was to either improve upon existing microinteractions or create my own, in this light i improved two existing microinteractions and created a new interaction for my personal portfolio website. These projects were all developed with HTML and CSS in a timeline of 9 weeks.

Context & Challenge:

The goal of this project was to learn to identify and implement the elements required to create incredible digital experiences. Through the application of user-experience design practices, I developed enjoyable interactive designs based on a consideration of feedback and users' experience. I was free to chose what original interface i wanted to improve the only restrictions were they had to have all the characteristics from the Microinteractions Model:

  • Triggers
  • Rules
  • Feedback
  • Loops & Modes

Process and Solutions:

Alpha

For the alpha I started off with a simple hoverstate. I noticed on the Drexel One student portal that the "schedule an appointment" button doesnt have a change in color or anything on hover. This made it an ideal candidate for a simple microinteraction. The improved microinteraction is comprised of the following:

  • Triggers: While the mouse hovers over the button it starts the microinteraction and when the mouse leaves the button the interaction stops.
  • Rules: While mouse hovers the button should change color and text color. A Drexel gold (#FFC600) and navy (#07294D) text should work well. When mouse stops hovering it should go back to the original color right away.
  • Feedback: On hover the user will see the button change color seamlessly from the current blue to gold and the text color change so there is solid contrast between button color and text color. It will blend between colors over time so it isnt just one color changing to the other.
  • Loops & Modes: Not applicable here.

Beta

For the beta I decided to improve an interaction on my personal portfolio website. In its current state my personal website does not contain a micro interaction within the header. Addionally it would be worth to add that on the creations page there is content related to multiple categories. The goal was to create a dropdown menu within the header that has a hover effect microinteraction and to seperate content into categories within that dropdown. The improved microinteraction is comprised of the following:

  • Triggers: Upon mouse hover over the Creations Nav button the microinteraction is triggered. Within the list another Hover effect occurs while mouse hovers over the Creations Nav tabs.
  • Rules: When mouse hovers color changes from black to grey and dropdown appears. The animation should fade in and do so over half a second. When dropdown appears user can hover over dropdown tabs. Upon hovering over the dropdown tabs the tabs change color.
  • Feedback: The user upon hovering over creations should see a color change where the mouse is hovering and alist appears beneath it. Within that list appears more options for navigation and if the user hovers over any of those tabs it will have a hover effect where the color changes too.
  • Loops & Modes: If the user hovers over Creations the tabs are set to appear. The data set to show are the various content pages. When there is no hover no extra data is displayed.
